# 部署指南：阿里云域名绑定教程 (Aliyun Domain Binding Guide)

针对你的域名 `lab107.asia`，以下是详细的阿里云 + Vercel 配置步骤。这样配置后，国内外用户都可以通过 `lab107.asia` 快速访问你的博客。

## 第一步：在 Vercel 添加域名
1. 打开 Vercel 项目控制台。
2. 点击顶部的 **Settings** (设置) -> 左侧菜单 **Domains** (域名)。
3. 在输入框中输入 `lab107.asia`，点击 **Add**。
4. Vercel 会自动建议添加 `www.lab107.asia`，也点击 **Add**。
5. 此时你会看到两个域名显示 "Invalid Configuration"（配置无效），因为我们还没去阿里云设置。Vercel 会给出需要的记录值（通常是 A 记录指向 `76.76.21.21`，CNAME 指向 `cname.vercel-dns.com`）。

## 第二步：在阿里云添加解析 (关键步骤)
1. 登录阿里云控制台，找到你的域名列表（就像你截图里那样）。
2. 点击 `lab107.asia` 右侧的 **解析** 按钮。
3. 点击 **添加记录**，你需要添加**两条**记录：

### 第 1 条：主域名 (@)
- **记录类型**：`A`
- **主机记录**：`@`
- **解析请求来源**：`默认`
- **记录值**：`76.76.21.21`
- **TTL**：`10分钟`

### 第 2 条：子域名 (www)
- **记录类型**：`CNAME`
- **主机记录**：`www`
- **解析请求来源**：`默认`
- **记录值**：`cname.vercel-dns.com`
- **TTL**：`10分钟`

> **注意**：如果阿里云提示 `@` 记录与 MX 记录冲突（通常是因为企业邮箱），你可以把第 1 条的记录类型也改为 `CNAME`，记录值填 `cname.vercel-dns.com`。但通常 A 记录更稳定。

## 第三步：等待生效
1. 回到 Vercel 的 Domains 页面。
2. 等待几分钟（通常不超过 10 分钟），原本红色的 "Invalid Configuration" 会变成蓝色的勾选图标。
3. 一旦变绿，恭喜你！现在访问 [https://lab107.asia](https://lab107.asia) 就可以在国内外流畅浏览了。

## 为什么这样能解决“被墙”问题？
Vercel 默认的 `*.vercel.app` 域名因为被太多人滥用，导致被 DNS 污染。而你自己的域名 `lab107.asia` 是干净的，直接解析到 Vercel 的服务器 IP (`76.76.21.21`) 通常是畅通无阻的。

## 进阶：国内外分流（可选）
如果你发现国内访问 Vercel 直连速度不理想，阿里云支持“解析请求来源”分流。
- **默认/海外**：指向 Vercel (`76.76.21.21`)
- **中国大陆**：如果你有国内服务器或 CDN，可以单独设置。
*对于个人博客，通常直接使用上述的标准配置已经足够解决访问问题。*
